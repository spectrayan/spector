Architecture Decision Record: Multimodal Memory Support

Context

Spector needs to support ingestion from multiple source types: text, images, audio, and video. The core cognitive engine stores only text and vectors, so non-text content must be transformed into text representations (captions, transcripts) before storage.

Decision

We will implement a SensoryExtractor SPI that transforms binary content into text chunks. The first implementation will use Ollama VLM for image captioning. Future implementations will support audio (via Whisper) and video (via FFmpeg + VLM).

Consequences

  • All non-text content is stored as text captions/transcripts plus URI pointers to original assets
  • The AssetStore SPI provides pluggable storage backends (local, S3, GCS)
  • Source modality is tracked in the binary header flags for SIMD filtering
